# Rook on Open File

Source: https://www.chessprogramming.org/Rook_on_Open_File

**[Home](/Main_Page "Main Page") \* [Evaluation](/Evaluation "Evaluation") \* [of Pieces](/Evaluation_of_Pieces "Evaluation of Pieces") \* Rook on (Semi) Open Files**

An [open file](/Open_File "Open File") is usually defined as a file with no pawns on it - a [semi-open file](/Half-open_File "Half-open File") as containing only the enemy pawns. In both cases a rook occupying such a [file](/Files "Files") gets greater vertical [mobility](/Mobility "Mobility") as well as a chance to penetrate the enemy camp, positioning itself on the [7th rank](/Rook_on_Seventh "Rook on Seventh").

Early versions of [Fruit](/Fruit "Fruit") omitted this evaluation term altogether. Some programs, like [Rebel](/Rebel "Rebel"), have an additional bonus for doubling rooks on an open file. Some increase the bonus if the file has a bearing against the [enemy king](/King_Safety "King Safety") , some make the bonus dependent on the number of own pawns. It is worth noting that [Nimzowitsch](https://en.wikipedia.org/wiki/Aron_Nimzowitsch) in [My System](https://en.wikipedia.org/wiki/Mein_System) used the term "(semi) open file" even for the rooks standing in front of the [pawn chain](/Pawn_Chain "Pawn Chain"). The same has been said about [Rybka](/Rybka "Rybka"). Bonuses applied to a rook on an open file vary from **8** to **20** [centipawns](/Centipawns "Centipawns") [[1]](#cite_note-1). Typical bonus for a semi-open file is half of that for a fully open file.

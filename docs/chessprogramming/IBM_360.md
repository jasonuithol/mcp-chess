**IBM System/360**,  
a 32-bit [mainframe computer](https://en.wikipedia.org/wiki/Mainframe_computer) system family announced by [IBM's](/index.php?title=IBM&action=edit&redlink=1 "IBM (page does not exist)") 2nd president, [Thomas Watson, Jr.](https://en.wikipedia.org/wiki/Thomas_Watson,_Jr.) on April 7, 1964, and delivered between 1965 and 1978. The chief architect of System/360 was [Gene Amdahl](https://en.wikipedia.org/wiki/Gene_Amdahl), the project was managed by [Fred Brooks](/Mathematician#FBrooks "Mathematician").

# Technology

Early 360 models used [Solid Logic Technology](https://en.wikipedia.org/wiki/IBM_Solid_Logic_Technology). The **360/195** (1971) had [ECL](https://en.wikipedia.org/wiki/Emitter-coupled_logic) [integrated circuits](https://en.wikipedia.org/wiki/Integrated_circuit) for the arithmetic and logic operations in the central processor combined with the 32 KiB high speed buffer memory. Each silicon memory chip was about 0.125 inches square and consisted of 664 [transistors](https://en.wikipedia.org/wiki/Transistor) and [diodes](https://en.wikipedia.org/wiki/Diode) etc. Each memory chip stored 64 bits of data. Two memory chips were mounted on a 0.5 inch ceramic substrate [[2]](#cite_note-2). A typical 360/195 configuration would be a single CPU with 2 MiB [core memory](/Memory "Memory"), an operator's [console](https://en.wikipedia.org/wiki/System_console) with a [cathode-ray tube](https://en.wikipedia.org/wiki/Cathode_ray_tube) (CRT) display for diagnostics and operator control [[3]](#cite_note-3).

# Architecture

The System/360 had a [pipelined](https://en.wikipedia.org/wiki/Pipeline_%28computing%29) architecture with 16 32-bit general-purpose registers, four 64-bit floating-point registers, and 24-bit (16 MiB) byte-addressable interleaved [memory](/Memory "Memory") of [Big-endian](/Big-endian "Big-endian") byte/word order. Most models used [microcode](https://en.wikipedia.org/wiki/Microcode) to implement the instruction set. Machine instructions had operators with operands, which could contain register numbers or memory addresses. Binary arithmetic and logical operations are performed as register-to-register and as memory-to-register/register-to-memory as a standard feature, and had 4-bit [nibble](/Nibble "Nibble") , 8-bit [byte](/Byte "Byte"), 16-bit halfword, 32-bit word , 64-bit double word or 128-bit quad word memory operands, or even 2048 byte storage blocks [[4]](#cite_note-4). Five separate units allowed the **360/195** to overlap and process up to seven different operations at the same time. The [floating-point](https://en.wikipedia.org/wiki/Floating-point_unit) unit could handle up to two additions and a multiplication at one time.

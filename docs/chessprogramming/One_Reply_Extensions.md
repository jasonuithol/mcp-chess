# One Reply Extensions

Source: https://www.chessprogramming.org/One_Reply_Extensions

**[Home](/Main_Page "Main Page") \* [Search](/Search "Search") \* [Selectivity](/Selectivity "Selectivity") \* [Extensions](/Extensions "Extensions") \* One Reply Extensions**

**One Reply Extensions** or **Singular Reply Extensions** are extensions applied if the [side to move](/Side_to_move "Side to move") has only one [legal move](/Legal_Move "Legal Move"). Most likely this extension is loss-seeking while in [check](/Check "Check") where [legal move generation](/Move_Generation#Legal "Move Generation") is used. Therefore One Reply Extensions are combined with [check evasion extensions](/Check_Extensions "Check Extensions"), often considered with additional [fractional plies](/Extensions#FractionalExtensions "Extensions") to extend. However, as already pointed out by [Bruce Moreland](/Bruce_Moreland "Bruce Moreland") [[1]](#cite_note-1) , there is some risk to create a sequence of check + single response that goes on essentially forever, which requires the One Reply Extension to be constrained.

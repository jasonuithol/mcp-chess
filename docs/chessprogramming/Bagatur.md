**Bagatur** (BagaturChess),  
a free [open source chess engine](/Category:Open_Source "Category:Open Source") by [Krasimir Topchiyski](/Krasimir_Topchiyski "Krasimir Topchiyski"), written in [Java](/Java "Java") and published under the [Eclipse Public License](https://en.wikipedia.org/wiki/Eclipse_Public_License) [[2]](#cite_note-2) at [GitHub](https://en.wikipedia.org/wiki/GitHub) [[3]](#cite_note-3).
The project has already started around year 2000. Under [Windows](/Windows "Windows") and [Linux](/Linux "Linux") the program hasn't its own [user interface](/User_Interface "User Interface") and uses [UCI](/UCI "UCI") to communicate with [GUIs](/GUI "GUI") such as [Arena](/Arena "Arena").
Under [Android](/Android "Android"), Bagatur comes with an own GUI, also incorporated as engine of [Chess Art For Kids](/index.php?title=Chess_Art_For_Kids&action=edit&redlink=1 "Chess Art For Kids (page does not exist)") [[4]](#cite_note-4).

# Etymology

[Bagatur](https://en.wikipedia.org/wiki/Baghatur) ([βαγατουρ](https://bg.wikipedia.org/wiki/%D0%91%D0%B0%D0%B3%D0%B0%D1%82%D1%83%D1%80) or βογοτορ) is a higher rank of [proto-Bulgarian](https://en.wikipedia.org/wiki/Bulgars) [warrior](https://en.wikipedia.org/wiki/Warrior). According to some authors the word comes from [Turkic](https://en.wikipedia.org/wiki/Turkic_languages) (bagadur) and literally means [brave](https://en.wikipedia.org/wiki/Brave), brave man. According to the same in the [Russian language](https://en.wikipedia.org/wiki/Russian_language) will pass as [Богатыри](https://ru.wikipedia.org/wiki/%D0%91%D0%BE%D0%B3%D0%B0%D1%82%D1%8B%D1%80%D0%B8) - [hero](https://en.wikipedia.org/wiki/Hero).

# Description

The programming code is structured in multiple Java [modules](https://en.wikipedia.org/wiki/Modular_programming) / [APIs](https://en.wikipedia.org/wiki/Application_programming_interface), which could be used separately, including modules for experiments with [artificial intelligence](/Artificial_Intelligence "Artificial Intelligence") and [machine learning](/Learning "Learning"). More details could be found inside the readme files of each sub-project of the programming code [[5]](#cite_note-5).
The [board is represented](/Board_Representation "Board Representation") as a mixture of [mailbox arrays](/Mailbox "Mailbox"), [pieces lists](/Piece-Lists "Piece-Lists") and [bitboards](/Bitboards "Bitboards").

## [Search](/Search "Search")

The main [interface](https://en.wikipedia.org/wiki/Interface_(Java)) of Bagatur's search [API](https://en.wikipedia.org/wiki/Application_programming_interface) is the bagaturchess.search.api.IRootSearch class, which is used directly from the IUCISearchAdaptor implementation located in the package bagaturchess.search.impl.uci\_adaptor.
The interface has three configurable implementations, which support either single [threaded](/Thread "Thread") search, [parallel search](/Parallel_Search "Parallel Search"), and a mixture of both in order to optimize search effectiveness for shallow depths.
All implementations are using the [MTD(f) algorithm](/MTD(f) "MTD(f)") algorithm based on the [PVS](/Principal_Variation_Search "Principal Variation Search") [alpha-beta](/Alpha-Beta "Alpha-Beta") search [[6]](#cite_note-6) [[7]](#cite_note-7).

## [Evaluation](/Evaluation "Evaluation")

Bagatur's evaluation consists of a [singe layer perceptron](/Neural_Networks#Perceptron "Neural Networks") so far, using the [Neuroph](https://en.wikipedia.org/wiki/Neuroph) [neural networks](/Neural_Networks "Neural Networks") Java framework.
[Supervised learning](/Supervised_Learning "Supervised Learning") by [value adaption](/Automated_Tuning#ValueAdaption "Automated Tuning") is used to [tune](/Automated_Tuning "Automated Tuning") the weights of the perceptron inputs,
featuring [material](/Material "Material"), [piece-square tables](/Piece-Square_Tables "Piece-Square Tables"), [mobility](/Mobility "Mobility"), [pawn structure](/Pawn_Structure "Pawn Structure"), and [king safety](/King_Safety "King Safety") beside many others.
Therefor, one has to feed in a huge number of [chess positions](/Chess_Position "Chess Position") with [evaluation scores](/Score "Score") given by a supervisor, for instance generated by Bagatur's **UCITracker** tool [[8]](#cite_note-8) along with strong chess engines like [Stockfish](/Stockfish "Stockfish") [[9]](#cite_note-9) [[10]](#cite_note-10).

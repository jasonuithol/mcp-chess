# Open File

Source: https://www.chessprogramming.org/Open_File

**[Home](/Main_Page "Main Page") \* [Chess](/Chess "Chess") \* [Files](/Files "Files") \* Open File**

According to the most obvious definition, an **Open File** is a vertical column with no [pawns](/Pawn "Pawn") of either [color](/Color "Color") on it. [Aron Nimzowitch](https://en.wikipedia.org/wiki/Aron_Nimzowitsch), however, stated in [My System](https://en.wikipedia.org/wiki/Mein_System) that there is something like a partially open file. For example, if there is a white pawn on h2, then the H-File is open for white from square h3 onwards. This definition has never caught up with chess programmers, probably because it would encourage an artificial piece placement.

A common [evaluation](/Evaluation "Evaluation") term associated with it is [rook on open file](/Rook_on_Open_File "Rook on Open File"). It may be worthwhile to use the same term in the queen evaluation, but the interaction with enemy rooks should be taken into account.
